01
COMPLETED
📊
Automated RAM Monitoring
& Email Alert System
Bash · crontab · ssmtp · Gmail SMTP

Production-grade Bash script monitoring server RAM every 5 minutes via crontab. When available memory drops below 300MB, it auto-sends a warning email to admin via Gmail SMTP (ssmtp) — preventing server crashes before they happen. Deployed on AWS EC2.

⚡ CHALLENGE SOLVED

Gmail blocked login with standard password. Fixed by enabling 2-Step Verification + generating a 16-digit Google App Password configured in /etc/ssmtp/ssmtp.conf. Also fixed inaccurate RAM data by using "Available" column (col 7) instead of "Free" in free -m.

BASH
AVAIL_RAM=$(free -m | awk 'NR==2{print $7}') if [ "$AVAIL_RAM" -lt 300 ]; then echo "$BODY" | mail -s "$SUBJECT" "$EMAIL" fi
BashcrontabssmtpGmail SMTPAWS EC2awkApp Password

🗄️
Log Archival & Storage
Optimization Script
find · gzip · cron · Error Handling

Intelligent Bash script scanning for files older than 10 days OR larger than 20MB, compressing them with gzip (up to 80% size reduction), and archiving them automatically. Runs daily at midnight via cron. Includes graceful error handling if no files match criteria.

⚡ PRODUCTION-GRADE DESIGN

Added -z empty check before the while loop — if FILES_LIST is empty, script exits with code 0 instead of running an empty loop or crashing. This is how real DevOps automation handles edge cases gracefully.

BASH
FILES=$(find "$DIR" -type f \( -mtime +10 -o -size +20M \)) if [ -z "$FILES" ]; then exit 0; fi gzip "$FILE" && mv "$FILE.gz" "$ARCHIVE"

Automated User Onboarding
& Account Creation Script
useradd · SHA256 · passwd -e · Root Check

Security-hardened Bash script for Linux server user management. Validates root privileges (UID=0), creates user accounts dynamically from arguments, auto-generates 12-char SHA256 random passwords from nanosecond timestamps, and forces password rotation on first login.

⚡ SECURITY FEATURES

Root UID check · Strong hashed passwords via sha256sum · Exit status validation ($? -ne 0) after every command · passwd -e forces immediate password change · shift command cleanly separates username from comment args.

BASH
PASSWORD=$(date +%s%N | sha256sum | head -c12) useradd -c "${COMMENT}" -m ${USER_NAME} echo "${USER_NAME}:${PASSWORD}" | chpasswd passwd -e ${USER_NAME} # Force change

Terraform · python-terraform · Dynamic AMI · One-Click Deploy

Provisioned a fully secure AWS EC2 instance using Terraform with dynamic AMI lookup (no hardcoded IDs), automated subnet selection, and a Python wrapper using python_terraform that orchestrates the entire init→plan→apply workflow with real-time terminal feedback — zero manual CLI needed.

⚡ MAJOR BLOCKER SOLVED

Hardcoded AMI IDs broke across AWS regions. Fixed with data "aws_ami" block querying latest Ubuntu 22.04 LTS dynamically. Also resolved subnet conflicts with data "aws_subnets" + depends_on for correct resource dependency ordering.

TERRAFORM
